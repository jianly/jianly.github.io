对HTTP、HTTPS的理解

2019.01.03 Janely HTTP  热度 ℃

基本常识

1、什么是TCP/IP协议

TCP/IP（传输控制协议）是指能够在多个不同网络间实现信息传输的协议簇。TCP/IP协议不仅仅指的是TCP和IP两个协议，而是指一个有FTP、SMTP、TCP、UDP、IP等协议构成等协议簇，只是因为TCP、IP具有代表性，所以被成为TCP/IP协议

2、什么是DNS

DNS(域名系统协议)，是一种分布式网络目录服务，主要用于域名与IP地址对相互转换，以及控制因特网对电子邮件对发送

3、什么是SSL/TSL

SSL(安全套接层)及其继任者TSL传输层安全是为网络通信提供安全及数据完整性对一种安全协议。TSL/SSL在传输层与应用直接对网络链接进行加密。

4、什么是URL/URI

http使用统一资源标识符（URI）来传输和建立连接。

  • URI: 统一资源标识符，用来标示一个具体的资源的，可以通过它知道一个资源是什么

  • URL: 统一资源定位符，用来定位具体资源，标示一个具体的资源位置

HTTP

超文本传输协议，是一个简单对请求响应协议，运行在TCP之上。

1、工作原理

基于客户端/服务器模式，且面向链接对，事务处理的过程：

  • 客户端与服务器建立tcp链接

  • 客户端向服务器发送请求命令，客户端发送请求头信息

  • 服务器接收请求，并根据请求返回响应的文件作为应答

  • 客户端与服务端关闭链接

2、报文格式

请求报文：

  • 请求行：包括方法、URL、协议/版本

  • 消息报头：request header

  • 请求正文

应答报文：

  • 状态行：版本、状态码

  • 消息报头

  • 响应正文

3、状态码

分类

  • 1xx：信息型，服务器收到请求，需要请求者继续操作

  • 2xx：成功型，请求成功收到，解析并处理

  • 3xx：重定向，需要进一步的操作完成请求

  • 4xx：客户端错误，请求包含语法错误或无法完成请求

  • 5xx：服务器错误，服务器在处理请求过程中发送了错误

常见状态码

  • 200：客户端请求成功

  • 301：资源被永久转移到其他URL

  • 302：临时跳转

  • 400：bad request 客户端请求有语法错误，不能被服务器所理解

  • 401：unauthorized 请求未经授权，这个状态码必须和www-authenticate报头域一起使用

  • 404：请求资源不存在，可能输入了错误的URL

  • 500：服务器内部发生了不可预期的错误

  • 503：服务器当前不能处理客户端的请求，一段时间后可能恢复正常

4、特点

  • 支持客户/服务器模式

  • 简单快速：请求服务时，只需要传送请求方法和路径

  • 灵活：允许传输任意类型的数据对象，传输类型由content-type标记

  • 无连接：限制每次链接只处理一个请求，服务器处理完请求，并收到客户的应答后，即断开连接，但是却不利于客户端与服务器保持会话连接，为了弥补这种不足，产生了两项记录http状态的技术，一个叫cookie，一个叫session

  • 无状态：协议对于事物处理没有记忆，后续处理需要前面的信息，则必须重传

5、请求方法

方法

  • get：向特定资源发起请求

  • post：向指定资源提交数据进行数据处理。数据包含在请求体中，post请求可能用于新的资源的创建以及已有资源的修改

  • delete：请求服务器删除指定资源

  • put：向指定资源上传最新内容

  • options：返回服务器对特定资源支持的http方法，用来向服务器发送‘*’的请求来测试服务器的功能性

  • head：类似与get请求，只不过没有返回体，用于获取包含在响应消息头中的元信息

get和post区别

  • get从服务器上获取资源，post向服务器发送数据

  • get传输数据是通过URL请求，以field（字段）= value的形式，置于URL后，并用”?”连接，多个请求数据间用”&”连接，post是通过http的post机制，将字段与对应值封存在请求实体中发送给服务器的

  • get传输的数量小，因为受URL长度限制，但是效率高。post可以传输大量数据

  • get不安全，因为URL可见，相对来说post比较安全

  • get只能支持ASCII字符，向服务器传的中文字符可能会乱码。post支持标准字符集，可以正确传递中文字符。

6、http1.0和1.1的区别

  • 长链接：在一个tcp连接上可以传送多个http请求和响应，减少了建立和关闭连接的消耗和延迟，在http1.1中默认开启connection：keep-alive，而http1.0每次请求都要创建连接

  • 缓存处理：1.1引入了更多的缓存控制策略如if-Match、if-NoneMatch等

  • 错误通知的管理：增加了24个错误状态响应码

  • 节约带宽：例如客户端只需要某一个对象的一部分，1.0中服务器将整个对象送过来，并不支持断点续传功能。1.1引入了range头域，允许只请求资源的某个部分

  • Host的头处理：1.0中认为每个服务器都绑定一个唯一的ip地址，但是一台物理服务器上可以存在多个虚拟主机，并且共享一个ip。

7、2.0和1.x的区别

  • 二进制分帧：http1.x的解析上基于文本，存在缺陷，因为文本的表现形式多样性，做到健壮性考虑的场景比较多。然后二进制只认0和1的组合

  • 多路复用：连接共享，即每一个request都是用左连接共享机制的。一个request对应一个id，这个一个连接上可以有多个request，每个连接的request可以随机混杂在一起，接收方可以根据request的ID将request再归属到各自不同的服务器请求里

  • 头部压缩：http1.x中头部带有大量信息，每次都要重复发送，2.0使用encoder来减少需要传输的header大小

  • 服务器推送

HTTPS

HTTPS在HTTP上建立SSL加密层，并对传输数据进行加密，是HTTP协议对安全版。

作用

  • 对数据进行加密，并建立一个信息安全通道，来保证传输过程中的数据安全。

  • 对网站服务器进行真实身份认证

HTTP/HTTPS区别

  • HTTPS可以有效的防止运营商劫持

  • http和https使用的是完全不同的连接方式，http是80端口，https是443端口

  • https需要认证证书

  • http运行在tcp之上，明文传输，https运行在ssl和tsl之上，ssl/tsl运行在tcp之上，且加密传输
