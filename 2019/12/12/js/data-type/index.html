JavaScript中常用数据类型

2019.12.12 Janely Javascript  热度 ℃

一、基本数据类型

1. Number(数字)

包括常规数字、NaN

NaN

not a number）不是一个数，但它率属于数字类型
NaN和任何值（包括自己）都不相等：所以不能用相等但方式判断是否有效数字

isNaN

检测一个值是否为非有效数字，如果不是有效数字返回true，反之是有效数字返回false

注意： 在使用isNaN进行检测时，首先会验证检测的值是否为数字类型，如果不是，先基于Number()方法把值转换为数字类型，然后再检测

把其他类型值转换为数字类型

a、 Number([val])
b、parseInt/parseFloat([val], [进制])，从左到右依次查找有效数字字符，直到遇到非有效数字字符。

  • 字符串转换为数字，空字符串会变为数字0
  • 布尔值转换为数字
  • null和undefined
  • 把引用数据类型转换为数字，是先把它基于toString的方法转化为字符串，然后再转换为数字(底层渲染规则)

2. String(字符串)

单引号，双引号、反引号包起来的都是字符串

其他类型转换为字符串

null和undefined是禁止直接toString方法的

  • [val].toString()

普通对象.toString()的结果”[object, object]”, 因为Object.prototype.toString方法不是转换为字符串的，而是用来检测数据类型的

  • 字符串拼接

四则运算法则中，除加法之外，其余都是数学计算，只有加法可能存在字符串拼接。（一旦遇到字符串，则不是数学运算，而是字符串拼接）

3. Boolean（布尔值）

只有两个值 true / false

把其他类型值转换为布尔值

只有0、NaN、’’、null、undefined 五个值转换为false，其余都转换为true。(而且没有任何的特殊情况)

  • Boolean([val])
  • !/!!, !先转为布尔值，然后取反
  • 条件判断

如果条件只是一个值，不是==/===/!=/>= 等这些比较，先转换为布尔值，再验证真假

4. null(空对象指针) / undefined(未定义)

null和undefined都代表没有

区别

  • null：意料之中，作为初始值，因为0不是空值，在栈内存中有自己都存储空间
  • undefined: 意料之外，创建一个变量没有赋值，默认是undefined

2. object-数组对象

数组是特殊的对象数据类型

  • 中括号中设置的是属性值，它的属性名是默认生成的数字，从零开始递增，而且这个数字代表每一项，称为索引 => 从零开始，连续递增，代表每一项位置的数字属性名
