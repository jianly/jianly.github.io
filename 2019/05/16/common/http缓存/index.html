HTTP缓存机制

2019.05.16 Janely http  热度 ℃

为什么要缓存

网络请求资源缓慢会影响用户体验，很多短期之内不会经常发生变化的资源文件没有必要每次访问都向服务端进行数据请求，缓存策略的使用就是为了改善客户端的呈现时间，降低服务端的负担。

呈现形式

策略体现在HTTP头部信息字段上，这些策略根据是否需要重新向服务端发起请求可以分为强缓存和协商缓存两大类。

强缓存（Expires、Cache-Control）

紧跟着一个缓存时间期限，当浏览器请求资源的时候会查看缓存中的资源是否存在并且确定该缓存的资源是否过期

缓存在保质期内

  • 客户端请求资源

  • 查看缓存反馈有需要的资源并且没有过期，返回资源数据

缓存超过保质期

  • 客户端请求资源

  • 查看缓存反馈数据已失效

  • 客户端再次向服务器请求资源

  • 服务器返回新的资源文件已经缓存机制

  • 将新的数据和缓存规则放入缓存系统

协商缓存

协商缓存无论什么情况都会和服务器交互，但是两种方式可以共同存在，强缓存的优先级较高，意味着请求一个资源时会先比较强缓存的字段，如果命中则不会再执行接下来协商缓存的过程。

缓存中命中

  • 客户端请求获取缓存资源的标识信息

  • 缓存返回该资源的标识信息

  • 服务器携带资源标识信息请求服务器比对缓存资源是否可用

  • 服务器返回资源可用信号

  • 客户端请求使用数据

  • 缓存反馈数据

缓存未命中

  • 客户端请求获取缓存资源的标识信息

  • 缓存返回该资源的标识信息

  • 客户端携带资源标识信息请求服务器比对缓存资源是否可用

  • 服务器发现资源已经改变

  • 服务器返回新的资源以及新的缓存标识

  • 客户端将新的数据和缓存规则放入缓存系统
