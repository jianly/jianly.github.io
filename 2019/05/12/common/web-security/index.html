XSS攻击

什么是XSS攻击

跨站脚本（Cross Site Scripting)，为了和 CSS 区分，故叫它XSS。XSS 攻击是指浏览器中执行恶意脚本(无论是跨域还是同域)，从而拿到用户的信息并进行操作。最常见的就是窃取Cookie

存储型

将恶意脚本存储到服务器数据库，然后在客户端执行这些脚本，从而达到攻击效果。常见例子：留言评论区提交恶意代码

反射型

指的是恶意脚本作为网络请求的一部分，通过作为请求的参数，经过服务器，然后再反射到页面中进行解析

文档型

并不会经过服务器，而是作为中间人到角色，在数据传输过程劫持到网络数据包，然后修改里面到html文档。

防范措施

1、利用CSP，CSP是浏览器中的内容安全策略，核心是服务器决定浏览器加载哪些资源

  • 限制其他域下的资源加载，并且禁止向其他域提交数据
  • 提供上报机制

2、利用httpOnly

cookie设置httpOnly后，js无法读取cookie

3、对用户输入对内容进行转码或者过滤

CSRF攻击

什么是CSRF

跨站请求伪造 (Cross-site request forgery)，指的是黑客诱导用户点击链接，打开黑客对网站，然后利用用户目前对登录状态发起跨站请求。

自动发起请求（GET / POST）

请求会自动带上xxx.com的cookie信息，如果服务器没有相应的验证机制，会默认该用户为正常用户，那就可以进行各种操作

防范措施

1、验证站点来源，要用到请求头中的两个字段: Origin和Referer，安全性差

2、CSRF Token，浏览器向服务器发送请求时，服务器生成一个字符串，将其植入到返回的页面中。

3、利用Cookie的SameSite属性

  • 在Strict模式下，浏览器完全禁止第三方请求携带Cookie。比如请求sanyuan.com网站只能在sanyuan.com域名当中请求才能携带 Cookie，在其他网站请求都不能。

  • 在Lax模式，就宽松一点了，但是只能在 get 方法提交表单况或者a 标签发送 get 请求的情况下可以携带 Cookie，其他情况均不能。

  • 在None模式下，也就是默认模式，请求会自动携带上 Cookie。
