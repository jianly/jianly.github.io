ES6、ES7、ES8新特性及使用方法的整理

2019.10.09 Janely Javascript  热度 ℃

ES6

let

  • 声明变量，用法类似于var，但是没有var的变量提升现象，只在命令所在的代码块内有效，形成了封闭作用域，凡在变量之前使用这些变量就会报错。
  • 暂时性死区

在代码块内，使用let命令声明变量之前，该变量都是不可用的。这在语法上，称为“暂时性死区”。ES6 规定暂时性死区和let、const语句不出现变量提升，主要是为了减少运行时错误，防止在变量声明前就使用这个变量，从而导致意料之外的行为。这样的错误在 ES5 是很常见的，现在有了这种规定，避免此类错误就很容易了。

  • 块级作用域

es5中只有全局作用域和函数作用域，没有块级作用域，这带来很多不合理的场景。

  1. 内层变量可能会覆盖外层变量
  1. 用来计数的循环变量泄漏为全局变量

const

  • 声明一个只读的常量，一旦声明，常量的值就不能改变。
  • 一旦声明变量，就必须初始化，不能留到以后赋值
  • 作用域与let相同，只在声明所在的块级作用域内有效
  • 声明的常量也没有变量提升现象，同样存在暂时性死区，只能在声明的位置后面使用
  • 声明的常量，不可重复

顶层对象和globalThis

  1. 顶层对象

顶层对象，在浏览器环境中指的是window对象，在NODE中指的是global对象。es5中顶层对象的属性与全局变量是等价的。

顶层对象的属性和全局变量挂钩带来的问题：

  • 没有办法在编译时就报出变量未声明的错误，只有运行时才能知道
  • 很容易误操作创建了全局变量
  • 顶层对象的属性是到处可以读写的，不利于模块化编程
  • window对象有实体含义，指的是浏览器的窗口对象，顶层对象是一个有实体含义的对象，也是不合适的

es6的规定：

  • 为了保持兼容性，var和function声明的全局变量，依旧是顶层对象的属性
  • let、const、class声明的全局变量，不属于顶层对象的属性
  1. globalThis

js中存在一个顶层对象，它提供全局环境，所有代码都是在这个环境中运行。但是，顶层对象在各种实现里面是不统一的

  • 浏览器中，顶层对象是window，但是node和web worker里没有window
  • 浏览器和web worker里，self也指向顶层对象，但node中没有self
  • node中，顶层对象是global，其他环境都不支持

一般使用this变量，但有局限性

  • 全局环境中，this会返回顶层对象，但是在node模块和es6模块中，this返回的是当前模块
  • 函数里的this，如果函数不是作为对象的方法运行，而是单纯作为函数运行，this会指向顶层对象，严格模式下，会返回undefined
  • new Function(‘return this’)()，总是会返回全局对象。但是，如果浏览器用了 CSP（Content Security Policy，内容安全策略），那么eval、new Function这些方法都可能无法使用。

解构赋值

从数组和对象中提取值，对变量进行赋值，这被称为解构

  • 数组
  • 对象

对象的解构与数组有一个重要的不同。数组的元素是按次序排列的，变量的取值由它的位置决定；而对象的属性没有次序，变量必须与属性同名，才能取到正确的值。

内部机制：先找到同名属性，然后再赋给对应但变量

  • 字符串但解构赋值
  • 数值和布尔值但解构赋值

如果等号右边是数值和布尔值，则会先转为对象

  • 函数参数但解构赋值

用途

  1. 交换变量但值
  2. 从函数返回多个值
  3. 函数参数但定义
  4. 提取json数据
  5. 函数参数但默认值
  6. 遍历map结构
  7. 输入模块但指定方法
