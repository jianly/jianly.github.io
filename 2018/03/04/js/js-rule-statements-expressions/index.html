我的JS规范三语句和表达式

2018.03.04 Janely JS编程规范  热度 ℃

一、块语句

不论块语句包含多行代码还是单行代码，都应当总是使用花括号。

  • if
  • for
  • while
  • do…while…
  • try…catch…finally

二、花括号都对齐方式

  • 将左花括号放置在块语句中第一句代码都末尾

  • 将左边花括号放置于块语句首行都下一行

三、块语句都间隔

第二种比较常见

四、switch语句

  • 缩进：
    • 每条case语句相对于switch关键字都缩进一个层级
    • 从第二条case语句开始，每条case语句前后各有一个空行
  • case 语句的“连续执行”

执行完一个case后连续执行下一个case。两种明显的“连续执行”：

  • default

switch语句中不应当省略default，如果default不需要执行任何，可以注释说明

五、ith语句

with语句可以更改包含的上下文解析变量当方式，虽然可以将对象前缀统统省略掉，但是js引擎和压缩工具无法对代码进行优化，因为它们无法猜出代码对正确含义。所以在严格模式下，with语句是被严格禁止对，应避免使用。

六、for循环

  • 传统的for循环，往往用于遍历数组

  • for-in循环，用来遍历对象

七、for-in循环

遍历对象属性，不用定义任何控制条件，循环会遍历每个对象的属性，并返回属性名而不是值。

for-in不仅遍历对象的实例属性，同样还遍历从原型继承来的属性。最好使用hasOwnProperty()方法来为for-in循环过滤实例属性
