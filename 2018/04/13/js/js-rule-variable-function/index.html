我的JS规范四变量、函数和运算符

2018.04.13 Janely Javascript  热度 ℃

一、变量声明

用var声明变量，在函数内部任意地方定义变量和函数顶部定义变量是完全一样的，因此所有的变量声明应该放在函数顶部而不是散落在各个角落。

函数声明

和变量声明一样，函数声明也会被js引擎提前。推荐先声明函数然后使用函数，函数内部的局部函数应当紧接着变量声明之后声明

三、函数调用间隔

一般情况下，对于函数调用写法推荐的风格是，在函数名和左括号之间没有空格，为了将它和块语句区分开。

严格模式

通过这种方式来谨慎地解析执行JS，以减少错误。不推荐在全局作用域中使用，因为这会让文件中所有代码都以严格模式来解析。

相等

由于js具有强制类型转换机制，会驱使某种类型都变量自动转换成其他不同类型，所以推荐使用===和!==来作比较。

  • 使用来判断相等运算符==和!=的时候，使用===和!==的时候
  • null、undefined

eval()

参数是一个字符串，会将传入的字符串当作代码来执行。严格模式对于eval()有着严格的限制，禁止在一个封闭的作用域中使用它创建新变量或者函数，这条限制帮助我们避免了eval()先天的安全漏洞（严格模式下，传入eval()的字符串无法在调用函数所在的上下文声明变量或函数）。特殊情况：将ajax的返回值转换为js值的场景下

原始包装类型

js中一个不易被了解且常常被误解的方面是，对原始包装类型的依赖。3种原始包装类型。禁止使用原始类型

  • String

  • Number

  • Boolean

每种类型都代表全局作用域中的一个构造函数，并分别表示各自对应的原始值的对象，主要作用是让原始值具有对象般的行为
